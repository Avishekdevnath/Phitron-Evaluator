# Phitron Extension - Icon Generation Prompt

## Overview
Create a cohesive set of icons for the Phitron Assignment Evaluator Chrome extension. All icons should follow a modern, clean design language with consistent styling.

## Icon Requirements

### 1. Extension Icon (Main Badge) - 128x128px
**Purpose**: Appears in Chrome toolbar, extension list, and Chrome Web Store

**Design Brief**:
- Modern, professional appearance
- Color scheme: Blue (#0066CC) as primary, white background
- Central symbol: Combination of:
  - Pencil/pen (representing assignments)
  - Star or checkmark (representing evaluation/completion)
  - AI spark or lightbulb (representing AI-powered)
- Style: Flat, minimal design with slight shadow
- Readable at small sizes (16x16px, 48x48px)
- Clean, modern sans-serif approach
- Should evoke: education, grading, intelligence

**Key Elements**:
- Should work in grayscale at 16px
- Must be recognizable in 32x32 size
- Professional enough for app store

**Variations Needed**:
- 16x16px (toolbar)
- 32x32px (toolbar retina)
- 48x48px (extension list)
- 128x128px (Chrome Web Store)

## Design System Guidelines

### Color Palette
```
Primary Blue:     #0066CC (Navigation, main actions)
Success Green:    #10B981 (Positive actions, activate)
Edit Orange:      #F59E0B (Edit, modify)
Delete Red:       #EF4444 (Destructive actions)
AI Purple:        #8B5CF6 (AI features)
Gray Neutral:     #6B7280 (Secondary, disabled)
Dark Gray:        #374151 (Active states)
Light Gray:       #F3F4F6 (Backgrounds)
```

### Style Requirements
- **Stroke Width**: 2px for line-based icons
- **Border Radius**: 2-4px for modern feel
- **Shadow**: Subtle drop shadow (optional) for depth
- **Consistency**: All icons should use same visual weight
- **Padding**: 2-4px internal padding in bounding box
- **Alignment**: Center-aligned, vertically balanced

### Export Formats Needed
- SVG (scalable, primary format)
- PNG 32-bit (16x16, 32x32, 48x48, 128x128)
- ICO format (for favicon/extension icon)

## Notes for Icon Designer

1. **Consistency is Key**: All icons should feel like they belong to the same family
2. **Scalability**: Must remain clear and recognizable from 16px to 128px
3. **Color Usage**: Use colors sparingly for action/state indication
4. **Negative Space**: Use whitespace effectively for clarity
5. **Alignment**: Center align all icons in their viewbox
6. **Testing**: Test all icons at actual sizes in the UI
7. **Animation Ready**: Design spinners to rotate smoothly
8. **Accessibility**: Ensure sufficient contrast ratios
